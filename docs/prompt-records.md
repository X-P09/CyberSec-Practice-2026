## Prompt 记录 #1 — 风险识别辅助

**使用场景：** 动手改代码前，用 AI 辅助梳理安全风险

**完整 Prompt：**

```
你是一位信息安全工程师。请审查以下 Python TCP 文件下载客户端代码，
识别其中存在的安全风险，并按严重程度（高/中/低）排序，
每个风险给出：漏洞类型、攻击场景描述、影响范围。

要求：
- 只分析安全问题，不要重写代码
- 重点关注：输入处理、文件系统操作、网络传输三个方面
- 如果某个风险在代码中已有部分防护但不完整，也要指出

[粘贴了 Client.py 完整代码]
```

**AI 响应关键片段（与安全约束相关）：**

> "发现以下安全风险：
> 1. **路径穿越（高危）**：`validate_filename()` 仅检查 `..` 字符，但未使用 `os.path.realpath()` 规范化路径，攻击者可通过 URL 编码 `%2e%2e` 或 Windows 风格 `..\\` 绕过校验...
> 2. **明文传输（中危）**：Socket 连接未加密，建议至少添加 HMAC 消息认证...
> 3. **写入内容未限制（中危）**：`receive_file()` 无文件大小上限，可导致磁盘耗尽..."

**偏差记录：** AI 同时建议"添加 TLS 支持"，但考虑到本项目为本地实验环境，已在约束文档中明确不引入新依赖，对此建议予以忽略，记录于 fix-report.md。

## Prompt 记录 #2 — 安全约束表达

**使用场景：** 将约束文档粘贴给 AI 后请求生成修复代码

**完整 Prompt：**

```
【项目约束文档——请在开始前完整阅读】
[粘贴了 constraint-doc.md 全文]

现在请基于以上约束，对下面的 Client.py 进行安全整改，
重点修复：
1. R-01 路径穿越：使用 os.path.realpath() + OUTPUT_DIR 白名单
2. R-03 接收内容未校验：添加 MAX_FILE_SIZE 100MB 限制 + 扩展名白名单
3. R-04 异常处理：将裸 except Exception 改为具体异常类型

请在每处修改旁添加注释说明修复的风险编号。
R-02（明文传输）本次暂不修复，仅添加警告注释。
禁止引入任何第三方库。

[粘贴了 Client.py 代码]
```

**AI 响应关键片段（安全约束落实情况）：**

> "已按照约束文档要求完成修复：
> - **R-01 修复**：在 `validate_filename()` 中新增 `os.path.realpath()` 规范化路径，并与 `OUTPUT_DIR` 对比，确保路径不超出允许目录
> - **R-03 修复**：新增 `ALLOWED_EXTENSIONS` 白名单集合 `{'.txt', '.pdf', '.py', '.json', '.md', '.csv'}`，`receive_file()` 开始前校验扩展名；新增 `MAX_FILE_SIZE = 100 * 1024 * 1024`，接收循环累计字节超限则中断并删除已写文件
> - **R-04 修复**：将 3 处裸 `except Exception` 改为 `except (socket.timeout, OSError) as e:`，安全相关异常（路径校验失败）改为 `raise`
> - 已在 R-02 相关的 `connect_server()` 处添加警告注释：'WARNING: 当前为明文传输，生产环境应使用 SSL/TLS 加密'"

**符合约束情况：** ✅ 全部约束得到落实，未引入第三方库，原有业务逻辑未被破坏。

## Prompt 记录 #3 — 审查验证

**使用场景：** 请 AI 对自己生成的代码进行二次审查

**完整 Prompt：**

```
请对以下修复后的 Client.py 进行安全审查：

检查项：
1. R-01 路径穿越防护是否完整？是否存在绕过可能？
2. R-03 文件大小限制是否在接收循环中正确累加？
3. 异常处理是否覆盖了所有关键路径？
4. 是否存在新引入的安全问题？

[粘贴修复后的代码]
```

**AI 审查发现（关键内容）：**

> "发现一个遗漏：`receive_file()` 在接收超限后删除临时文件时，未检查 `os.remove()` 的异常，若删除失败会留下不完整的大文件。建议在删除操作外包裹 `try/except OSError`。"

**处置：** 已根据审查意见补充 `os.remove()` 的异常处理，详见 fix-report.md。
