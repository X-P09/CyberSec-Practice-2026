# 项目约束文档

## 项目背景

这是《信息安全管理》课程的实践作业，仓库为 `renyanbinbin/CyberSec-Practice-2026`。  
我们正在对 `成员代码/xiezhizhuo/Client.py`（一个基于 TCP 的文件下载客户端）进行安全整改，已完成风险识别，现在需要 AI 协助编写修复代码。

---

## 强制约束条件（AI 必须遵守）

### 1. 路径穿越修复约束

- **必须** 使用 `os.path.realpath()` 规范化接收到的文件路径
- **必须** 验证最终路径在允许的输出目录（`OUTPUT_DIR`）范围内
- **禁止** 仅靠检查 `..` 字符串来防护路径穿越
- **禁止** 使用用户输入的原始字符串直接构造文件路径

### 2. 文件写入安全约束

- **必须** 使用 `with open()` 语句（上下文管理器）处理文件，确保文件句柄安全关闭
- **必须** 添加文件大小限制（建议 `MAX_FILE_SIZE = 100 * 1024 * 1024`，即 100MB）
- **必须** 对接收文件的扩展名做白名单校验（允许 `.txt`、`.pdf`、`.py`、`.json`、`.md`、`.csv`）
- **禁止** 接收并写入无扩展名或不在白名单内的文件

### 3. 异常处理约束

- **必须** 捕获具体异常类型（`socket.timeout`、`OSError`、`ConnectionRefusedError` 等）
- **禁止** 使用裸 `except Exception` 吞掉所有异常而不重新抛出
- **必须** 为安全相关异常（路径校验失败、文件超限）添加明确的日志记录

### 4. 代码风格约束

- **必须** 保持原有功能逻辑不变（只加安全层，不重写业务逻辑）
- **必须** 用语义化常量替代魔法数字
- **禁止** 引入新的第三方依赖库（只用 Python 标准库）
- **禁止** 使用 `eval()`、`exec()` 或 `subprocess` 处理任何输入数据

### 5. 文档约束

- **必须** 在修复的函数/逻辑处添加说明注释，标注修复了哪个安全风险（引用 R-01~R-04 编号）

---

## 禁止行为

- ❌ 禁止移除现有的文件名合法性校验逻辑（只能加强，不能删除）
- ❌ 禁止关闭 SSL/TLS 校验（本次虽未启用 TLS，但不得在注释中建议关闭验证）
- ❌ 禁止将用户凭证、密钥硬编码到代码中
- ❌ 禁止生成在正式环境中会打印敏感信息（文件内容、路径）的 DEBUG 日志

---

## 期望产出

1. 修复后的 `Client.py`，包含路径穿越防护 + 文件大小限制 + 扩展名白名单
2. 修复说明注释（每处修改标注原因）
3. 不破坏原有 TCP 连接和文件接收逻辑
